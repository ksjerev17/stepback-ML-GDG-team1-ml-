# Step Back ML — 전체 자체 점검 (효율성·타당성·신뢰성·차별성·재미)

> v9.6 기준. "내 파트(ML)의 모든 기능이 5개 축에서 어떤 가치를 주는지, 무엇을 강화했는지, 남은 한계는 무엇인지"를 정직하게 정리.

## 0. ML 기능 전체 지도

| 기능 | 모듈 | 한 줄 |
|---|---|---|
| 13차원 라벨링 | `labeler` + `llm_client` | 입력 → 인지6·행동2·감정5 점수 (LLM 또는 Mock) |
| 후처리 보정 | `label_postprocess` | 길이 기반 confidence 보정 + 환각 가드 |
| 추천 라우팅 | `recommender` | 위기→인지→행동→감정→약신호 단계 + 항상 드릴 |
| 설명가능 추천(why) | `recommender._build_why` | "왜 이 드릴인지" 자연어 + 근거 factor |
| 개인화(할인 UCB) | `personalization` | 선택을 누적·감쇠해 카테고리 가산점 |
| 주간/월간 코칭 | `weekly_coaching` | 상태 추론 + 다변량 경향 + 다음 초점 |

## 1. 효율성 (Efficiency)

**강점**
- LLM 호출 1회/일 + temperature 0.1(결정적) + max_tokens 400 → 비용·지연 최소.
- 개인화는 무거운 모델 없이 SQLite 카운터 + UCB 공식 → 메모리·CPU 거의 0.
- 드릴 카탈로그 `lru_cache`, 개인화 `bonus_map`은 카테고리 6개를 **DB 1회 조회**로 일괄.
- Mock 폴백으로 키 없이도 전 기능 동작 → 개발·CI 빠름.

**v9.6 강화**
- 할인 UCB로 통계가 무한 누적되지 않음(감쇠) → DB 행이 작게 유지.
- LLM 재시도에 지수 백오프 상한(4s) → 장애 시 무한 대기 방지.

**남은 한계 (정직)**
- 주간/월간 리포트는 호출마다 재계산. 표본이 7~30개라 비용은 작지만, 사용자 폭증 시 BE가 야간 배치로 미리 계산·캐시하면 좋음(스케줄러는 BE 영역 — `v96_personalization_handoff` 참고).

## 2. 타당성 (Validity)

**강점**
- 모든 설계가 학술 근거에 1:1 매핑 (Beck 인지왜곡 6 / Jacobson 행동활성화 / Hayes ACT / Iyengar 선택과부하 / Fogg 습관 / Neff 자기자비 / DBT 그라운딩 / **Auer 1985 UCB**).
- evidence_span으로 "판단의 출처"를 사용자 원문에서 인용 → 사후합리화가 아님.
- baseline 비교로 "평소 대비" 상대 판단 → 절대 점수의 임의성 보완.

**v9.6 강화**
- **할인 UCB(discounted UCB / Sutton & Barto 2.5)**: 개인화가 최근 선택을 더 반영 → "옛 취향에 갇히는" 편향 제거. 변하는 사람을 따라감.
- **다변량 경향**: 단일 변수(수면↔패턴)에서 수면추세·사교↔컨디션·컨디션↔행동·맥락↔표현 등으로 확장. 단, **전부 상관(correlation) 관찰 톤** — 인과 단정 0.
- **최소표본 가드**: 하루치 버킷으론 경향을 만들지 않음(잡음 차단). 강도에 따라 "뚜렷한 관찰/관찰/약한 관찰" 라벨로 과신 방지.

**남은 한계**
- 드릴이 "실제로 효과 있었는가"는 helpful(자기보고)로만 측정. 전/후 기분 측정은 FE 입력이 필요해 이번 범위에서 제외(추후 옵션).

## 3. 신뢰성 (Reliability)

**강점**
- 위기 4중 차단(한국어16+영어20+은유+띄어쓰기 우회) → **LLM 호출 전** 차단, 평문 외부 전송 0.
- PII 6종 마스킹 + 감사로그 평문 0%(SHA-256+salt) + 진단어 CI lint.
- LLM 파싱 실패·예외·빈응답 → 전부 Mock 폴백. 사용자에게 에러 노출 0.
- 결정적(temp 0.1 + JSON) → 같은 입력에 같은 출력.

**v9.6 강화**
- **항상 드릴 보장**: 전 카테고리가 거부돼도 카탈로그에서 안전하게 드릴 반환. 카탈로그가 비는 배포 사고에만 부드러운 skip.
- **LLM 재시도+백오프**: 일시적 레이트리밋/타임아웃을 흡수.
- **개인화 동시성**: `busy_timeout` + last-write-wins로 멀티 워커 잠금 충돌 완화.
- **빈/부분 입력 견고성**: 빈 label, 부분 context로도 안 깨짐(테스트로 고정).

**남은 한계**
- 실 Gemini 부하 시험은 GCP 기동 후 가능(코드·배포 스크립트는 준비). 현재는 Mock 동시성으로 검증.

## 4. 차별성 (Differentiation)

**GPT 단독 대비**
- **설명가능 추천(why)**: "어젯밤 잠이 부족해서, '미래예측'이 자주 보여서 골랐어요" — 근거를 사용자 언어로. (GPT는 "종합적으로요")
- **결정성**: 같은 입력 → 같은 응답. GPT는 매번 다름.
- **누적·개인화**: 주간/월간 통계 + 할인 UCB로 사람마다 다르게 적응.
- **시스템이 사람과 함께 변함**: 코칭의 "상태·경향·다음 초점"이 매주 갱신, 추천이 선택을 따라 이동.
- **안전 우선 구조**: LLM은 점수 "엔진", 사용자에게 말하는 "화자"는 결정적 함수 — 검증 한 겹 더.

**v9.6 강화**: why·다변량 경향·할인 UCB가 모두 "이 앱은 나를 관찰하고 나에게 맞춰 변한다"는 차별 체감을 키움.

## 5. 재미 / 지속사용 (Fun / Retention)

**강점**
- 매일 "내 말 인용 + 왜 이 드릴인지" → 들어주는 느낌.
- 주간 코칭의 상태 배지("회복기"), 경향 발견("잠 못 잔 날 미래예측↑"), 다음 초점 → 다음 주가 궁금해짐.
- 개인화 노트("지금까지 자기자비가 가장 잘 맞으셨어요") → 성장·내 것이 된 느낌.

**v9.6 강화**
- 다변량 경향이 많아져 매주 "새로운 발견"이 생김 → 재방문 동기.
- 할인 UCB로 추천이 점점 "나에게 맞아가는" 체감.

**남은 한계 / 의도적 선택**
- 스트릭·뱃지 같은 강한 게이미피케이션은 **일부러 넣지 않음**. 정신건강 인접 도구에서 "못 하면 죄책감"은 역효과. "오늘 안 해도 괜찮아요" 톤 유지 — 이 자체가 차별·타당성 포인트.
