# UIUX 팀원에게 보낼 메시지 (그대로 복사 → 전송)

---

안녕하세요! 와이어프레임 잘 봤어요 🙂 ML 기준으로 디자인할 때 참고할 점 + 살짝 바뀐 점 공유드려요.

**1. 드릴 카드 — "왜 이 드릴인지"를 3층으로 (핵심 차별 요소)**
와이어프레임의 드릴 카드("왜 이 드릴인지" 박스)가 우리 차별점이라, 정보를 3층으로 보여주면 좋아요:
- ① **왜** — "어젯밤 잠이 부족해서, '망할 것 같아'가 보여서 골랐어요" (사용자 원문 인용 포함)
- ② **도움** — "꼬리를 무는 생각에서 한 걸음 떨어져 보는 데 도움이 돼요" (효용)
- ③ **근거** — "CBT 생각 재구성 기법 · Beck(2020)" (작게, 신뢰감)
- 특히 ①의 **사용자 원문 인용**을 시각적으로 강조(따옴표/하이라이트)해주세요. "내 말을 읽었구나"가 핵심 감정이에요.

**2. 칩(태그) 디자인 — 두 종류를 시각적으로 구분**
- **사실 칩**(사용자가 입력): "수면 부족 · 4시간", "컨디션 2/5", "사교 갈등" → 진한 색.
- **해석 칩**(시스템 판단): 패턴 라벨. **임상용어 대신 일상어**로요 — "미래예측"❌ → "미리 결론 내리기"✅. 딱지 느낌 줄이게 연한 톤/테두리만.

**3. 드릴 색상 = 카테고리 6색 (드릴마다 다른 색 X)**
| 카테고리 | 제안 hex | 느낌 |
|---|---|---|
| 생각 다시 보기(인지) | `#F59E0B` 주황 | 환기 |
| 작은 행동 | `#10B981` 초록 | 활력 |
| 습관·루틴 | `#14B8A6` 청록 | 안정 |
| 지금·진정(그라운딩) | `#38BDF8` 하늘 | 차분 |
| 자기 자비 | `#F472B6` 분홍 | 따뜻 |
| 수면·리듬 | `#6366F1` 남보라 | 휴식 |
색만으로 구분하지 말고 아이콘/라벨 병기(접근성). 자세한 표 `drill_color_spec.md`.

**4. 위기 카드 (CrisisResultView)**
와이어프레임 좋아요. 따뜻하지만 분명하게 — "지금 많이 힘드신 것 같아요" + 상담 3개(1393/1388/1577-0199) + [전화 걸기]. 드릴 팔레트와 무관한 차분한 톤. (위기 시엔 드릴 추천 안 하고 이 카드만)

**5. 주간 리포트 — 경향 중심**
- **메인**: "이런 경향이 보였어요" 불릿 (신뢰도 라벨: 뚜렷한 관찰/관찰/약한 관찰 → 살짝 다른 강조).
- 상태("소진기")는 큰 진단 배지보다 **한 줄 요약** 위주로 부드럽게. "다음 주엔 ~" 은 지시가 아니라 **제안** 톤.
- 기록 적은 주는 "며칠 더 모이면 보여드릴게요" 빈 상태 1종.
- 감정 오각형 유지, **주 전반/후반 비교 차트 생략**.

**6. 톤 & 안전**
"못 하면 죄책감"이 아니라 "오늘 안 해도 괜찮아요". 스트릭/벌점 같은 압박 요소는 지양(정신건강 인접 도구).

**7. DrillLoadingView**
"오늘의 나에게 적합한 드릴을 찾는 중..." 좋아요. 실제 Gemini 호출이라 1~2초 걸릴 수 있어 로딩 화면 유지가 자연스러워요.
