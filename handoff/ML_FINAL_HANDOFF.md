# Step Back ML — 최종 인계 문서 (v9.7)

> 내 파트(기획+ML) 전체 완성 사항. ERD·와이어프레임 기준으로 각 파트가 뭘 받고 뭘 해야 하는지 한 곳에 정리. 상세 역할별 메시지는 `MESSAGE_to_FE/BE/UX.md` 참고.

---

## 1. 한 줄 요약
사용자가 매일 **텍스트 + 맥락(컨디션·수면·사교·운동)**을 입력하면 → ML이 **13차원 라벨링 → 드릴 추천(왜·효용·근거·원문인용)**을 돌려주고, 일주일이 쌓이면 **개인 맞춤 주간 리포트**(상태·다변량 경향·다음 초점)를 만든다. 최종 산출물은 **Gemini** 사용.

## 2. 완성된 ML 기능 전체

| 기능 | 설명 | 사용자가 받는 것 |
|---|---|---|
| 13차원 라벨링 | 인지6·행동2·감정5 점수 + evidence_span (Gemini, 키 없으면 Mock) | (내부) |
| 드릴 추천 | 100개 카탈로그, 임상 라우팅 + 개인화 가산 | 드릴 카드 |
| **설명가능 추천(why)** | 왜 골랐는지 + 효용 + 근거 + **원문 인용** | "왜 이 드릴인지" 3층 |
| 항상 드릴 정책 | drill / crisis_card 2종만 (잔잔한 날도 drill+tone:positive) | 단순한 카드 |
| 위기 안전망 | LLM 호출 전 차단, 상담번호 안내 | 위기 카드 |
| 개인화(할인 UCB) | 완료·도움·거부 학습, 최근 선택 가중 | 점점 맞춰지는 추천 |
| **주간/월간 코칭** | 상태 + 다변량 경향 + 다음 초점 + 개인화 노트 | 주간 리포트 |

## 3. 100개 드릴 (v6.4)
- 기존 77 + 신규 23 = 100. 카테고리 6종 분포: 인지30·행동26·습관17·그라운딩13·자기자비8·수면6.
- 신규 23개는 **갭 집중**: 과잉일반화·독심술·당위진술·이분법·분노·사교/수면 맥락.
- 100개 전부 **양식 통일**(behaviors 키) + **구체적 논문 근거**(저자·기법·효과, 평균 52자) + **3회 검증 통과**.
- 정본: `w1/drills_seed_v6_4.json`. 색상은 카테고리 6색 → `handoff/drill_color_table.json`.

## 5. 주간 리포트 개인 맞춤형 — 100% 사용자 입력 기반
사용자가 매일 넣은 것 → 주간 리포트로 변환:
- 매일 **컨디션** → 상태("소진기") + 흐름
- 매일 **수면** → "수면 점점 줄어드는 흐름 (7.3→4.8h)"
- **컨디션 ↔ 텍스트 라벨** → "컨디션 낮은 날 미래예측↑"
- **사교 ↔ 컨디션** → "갈등 날 컨디션 더 낮음 (2.5 vs 4.0)"
- 내가 쓴 **문장** → "힘든 날 '망할 것 같아' 표현 자주" (원문 인용)
- **운동 ↔ 컨디션** → "운동한 날 컨디션 더 좋음"
- 개인화 학습 → "지금까지 ~가 잘 맞으셨어요"

각 경향엔 신뢰도(뚜렷한 관찰/관찰/약한 관찰)가 붙고, 표본 부족 시 경향을 만들지 않음(잡음 차단).

## 6. ERD 점검 — BE가 추가할 것 (맞춤형의 전제)
| 필요 | ERD 위치 | 상태 |
|---|---|---|
| 컨디션 | `entries.self_condition` | ✅ 있음 |
| 수면·사교·운동 | `entries.context_json` | 🔴 **추가 필요** |
| 그날 13차원+evidence | `entries.label_result_json` | 🟡 저장 확인 |
| 완료·도움 | `entries.drill_completed`, `entries.helpful` | 🔴 **추가 필요** |
| 개인화 통계 | (ML 자체 SQLite) | ✅ BE 불필요 |

→ 와이어프레임에 입력 UI(수면/운동/컨디션/사교)는 이미 있음. **저장 연결만** 하면 주간 맞춤형이 다 동작. 상세 SQL: `weekly_erd_check.md`.

## 7. 실행 (Gemini 기본)
