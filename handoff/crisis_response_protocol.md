# 위기 응답 프로토콜 (전원) — v9.5

> 출처: CLAUDE.md §16.2. **변형 절대 금지**.

## 1. 위기 감지 (v9.5 확장)

ML 측 `app/infra/llm_client.py`의 `_detect_crisis()` 함수가 3 단계로 감지:

1. **한국어 직접 표현**: "죽고 싶", "사라지고 싶", "끝내고 싶", "자해", "목숨", "더 이상 살", "살고 싶지 않", "자살"
2. **한국어 은유**: "사라지면", "없어지면", "끝내버리", "이 세상에서 사라", "다 끝내", "포기하고 싶", "더는 못", "그만하고 싶"
3. **영어 표현**: "kill myself", "want to die", "wanna die", "end it all", "suicide", "self-harm", "hurt myself", "disappear forever", "don't want to live", "no point living", "thinking about dying" 등
4. **띄어쓰기 우회**: "죽 고 싶 어" 등 공백 우회도 감지 (compact matching)

LLM이 `CRISIS_DETECTED` 토큰 반환하거나 위 패턴 매칭되면 → 모든 점수 0 + `crisis_detected=true`.

## 2. 흐름

1. 사용자 텍스트 → ML이 위기 감지
2. ML이 `crisis_card` 반환 — 일반 드릴 절대 X (라우팅 Step 1 우선)
3. BE는 받은 카드를 **그대로** FE로 전달. 카피·전화번호 변경 절대 X
4. FE는 즉시 카드 표시 — 일반 드릴 카드와 시각적으로 명확히 구분 (다른 색·아이콘·레이아웃)
5. "전화 걸기" 버튼 → OS 다이얼러 호출 (`<a href="tel:1393">`)
6. ML / BE는 "카드 표시" 사실만 익명 로그 (사용자 텍스트 평문 X — audit_log가 hash + 요약만)


## 4. 절대 금지

- 일반 드릴로 라우팅
- 카피 변경 ("힘내세요", "이겨내세요" 등 응원 표현 X)
- 사용자 텍스트 평문 저장/로깅 (§11.5)
- 진단 표현 ("우울증입니다", "자살 위험군" 등)
- BE/FE 추가 카피 ("도움이 필요하시면" 등 임의 첨부 X)

## 5. 시연 시 안전

- 시연 입력에 위기 표현 사용 시 **화면 상단 워터마크 "시연 예제 — 실제 데이터 아님"** 명시
- 1393 단축번호는 시연 중 절대 누르지 않게 안내
- 발표자 폰: 와이파이 + 1393 단축번호 비활성화

## 6. P1·P2 합의 필요 (미해결)

- **위기 응답 직후 quota 면제 정책** — 위기 사용자가 추가 입력 시 429로 막히면 위험. 현재는 동일 quota.
  - 옵션 A: 위기 응답 후 다음 1회는 quota 면제
  - 옵션 B: 위기 응답 시점부터 ADMIN_TOKEN 자동 알림 → 운영자 수동 reset
  - 옵션 C: 현재 그대로 — quota 막혀도 BE entries는 저장됨 (단지 ML 라벨링만 X)
- **위기 응답 후 사용자 후속 행동** — BE/FE가 어떻게 안내? (FE 영역 — 별도 화면 또는 카드 fixed?)
